How should the /collections page be structured?

All 3 approaches are fully managed in the Shopify customizer. The difference is how the page is built.

A

Stack existing sections (no new code) ⭐ Recommended

Replace list-collections.json to stack existing sections in sequence in the customizer:

  • Featured Collection → Oversized Bags (existing section)
  • Rich Text / Image-with-Text → Promo banner (existing section)
  • Featured Collection → Mid Bags
  • Rich Text / Image-with-Text → Promo banner
  • Featured Collection → Clutch Bags

Pros

  • Zero new Liquid files
  • Fully flexible — add/remove/reorder in customizer
  • Reuses battle-tested theme code

Cons

  • Promo banner styling may need minor CSS tweaks to match the 3-column text+image+image layout
B

New "Collection Showcase" section

Build one self-contained custom section that includes: collection header + product grid + optional promo block. Merchant adds one section per collection category.

  • Collection Showcase → Oversized (products + promo block built-in)
  • Collection Showcase → Mid Bags
  • Collection Showcase → Clutch Bags

Pros

  • Clean, logical customizer UX per collection
  • Banner is always paired with its collection

Cons

  • Requires a new section file
  • Less flexible — promo banner is locked to each collection section
C

Redesign main-list-collections to show products

Modify the existing main-list-collections.liquid to loop through collections and render products for each, instead of collection tiles.

  • Auto-renders all collections with products inline
  • Promo banners would need a different mechanism (e.g. metafields)

Pros

  • Automatic — no manual setup per collection

Cons

  • Much harder to customize per-collection
  • Promo banners are awkward to insert
  • Not aligned with "use existing blocks" goal
